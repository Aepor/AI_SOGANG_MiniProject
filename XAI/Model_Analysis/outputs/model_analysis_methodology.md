# 모델 분석 방법론

## 분석 목적

이 프로그램은 `outputs_json`에 들어 있는 모델별 XAI 결과를 이용해 어떤 모델이 어떤 설명 특성에서 강한지 비교합니다.
정답 라벨이 포함되어 있지 않기 때문에 정확도, 재현율, F1-score 같은 분류 성능을 계산하지 않습니다.

## 비교 범위

- 최종 모델 비교에는 모든 모델에 공통으로 존재하는 방법만 사용합니다: **Integrated Gradients 100 steps, Integrated Gradients 50 steps, Occlusion**.
- 최종 모델 비교에는 모든 모델과 공통 XAI 방법에 모두 존재하는 입력 문장만 사용합니다. 현재 공통 입력 문장 수는 **30개**입니다.
- 특정 모델에만 있는 방법은 `method_strength_summary.csv`에서 참고용으로 확인합니다.
- 예를 들어 FNN의 LIME은 FNN 내부 설명 특성을 보는 데는 유용하지만, Transformer에 대응 방법이 없으면 최종 모델 순위에는 넣지 않습니다.
- Top-3 기반 지표는 단어 수가 4개 이상인 문장에 대해서만 계산합니다.

## 수식 기호

- `i`: 문장 또는 사례 인덱스입니다.
- `j`: 문장 안의 단어 인덱스입니다.
- `s_ij`: i번째 문장의 j번째 단어에 부여된 XAI score입니다.
- `|s_ij|`: score의 절대값이며, 설명량의 크기로 사용합니다.
- `p_i(y_hat_i)`: i번째 문장에서 모델이 예측 클래스에 부여한 확률입니다.
- `direction(y_hat_i)`: 예측이 positive이면 양수, negative이면 음수 방향입니다.
- 각 지표는 먼저 문장 단위로 계산한 뒤, 모델별 평균을 냅니다.

## 지표별 순위 산정

### 예측 확신도

- 의미: 모델이 자신이 예측한 클래스에 부여한 평균 확률입니다.
- 순위 기준: 평균 예측 확률이 높은 모델을 우수 모델로 봅니다.
- 간단 수식: `confidence = mean_i p_i(y_hat_i)`
- 차용한 방식: 모델의 softmax/sigmoid 예측 확률을 평균내는 방식입니다.
- 해석 메모: 정답 여부가 아니라 모델이 자기 예측에 부여한 확신을 봅니다.

### 예측 방향-설명 정합도

- 의미: 예측 감성과 같은 방향의 attribution score가 전체 설명량에서 차지하는 비율입니다.
- 순위 기준: 예측 방향과 같은 부호의 설명량 비율이 높은 모델을 우수 모델로 봅니다.
- 간단 수식: `alignment_i = sum(|s_ij| for sign(s_ij)=direction(y_hat_i)) / sum_j |s_ij|`
- 차용한 방식: signed attribution의 부호가 예측 감성 방향과 일치하는 비율을 봅니다.
- 해석 메모: positive 예측은 양수 score, negative 예측은 음수 score를 예측을 지지하는 설명으로 계산합니다.

### 핵심 단어 집중도

- 의미: 가장 큰 절대 score를 가진 단어 하나가 전체 설명량에서 차지하는 비율입니다.
- 순위 기준: 핵심 단어를 더 뚜렷하게 잡아내는 모델을 우수 모델로 봅니다.
- 간단 수식: `focus_i = max_j |s_ij| / sum_j |s_ij|`
- 차용한 방식: L1 정규화된 attribution에서 가장 큰 단일 단어의 비중을 보는 방식입니다.
- 해석 메모: 값이 높을수록 한 핵심 단어에 설명이 선명하게 집중됩니다.

### 유효 단어 커버리지

- 의미: 문장 내 단어 중 전체 설명량의 2% 이상을 차지한 단어의 비율입니다.
- 순위 기준: 의미 있는 크기의 설명 신호를 더 많은 단어에 부여한 모델을 우수 모델로 봅니다.
- 간단 수식: `coverage_i = count_j(|s_ij| / sum_k |s_ik| >= 0.02) / n_i`
- 차용한 방식: L1 정규화된 attribution에서 최소 비중 이상을 가진 단어만 세는 방식입니다.
- 해석 메모: 아주 작은 noise성 score를 제외하고, 전체 설명량의 2% 이상을 차지한 단어만 유효 설명 단어로 봅니다.

### 상위 3개 단어 집중도

- 의미: 절대 score 기준 상위 3개 단어가 전체 설명량에서 차지하는 비율입니다.
- 순위 기준: 중요 단어 3개에 설명이 선명하게 모이는 모델을 우수 모델로 봅니다.
- 간단 수식: `top3_share_i = sum(top3_j |s_ij|) / sum_j |s_ij|, only if n_i > 3`
- 차용한 방식: 상위 k개 중요 특성의 attribution mass를 보는 feature concentration 방식입니다.
- 해석 메모: 현재 k=3로 설정되어 있으며, 단어 수가 3개 이하인 문장은 평균에서 제외합니다.

### XAI 방법 간 일관성

- 의미: 같은 모델 안에서 여러 XAI 방법이 비슷한 핵심 단어를 고르는 정도입니다.
- 순위 기준: Top-3 단어 겹침과 절대 score 순위 상관이 높은 모델을 우수 모델로 봅니다.
- 간단 수식: `method_agreement = mean(top3_jaccard, spearman_abs_rank), only if n_i > 3`
- 차용한 방식: Top-k 집합 유사도(Jaccard)와 절대 score 순위 상관(Spearman)을 결합합니다.
- 해석 메모: Top-3 비교가 전체 단어 비교가 되지 않도록 단어 수가 3개 이하인 문장은 제외합니다.

## 산출 파일 해석

- `category_winners.csv`: 평가 부문별로 어느 모델이 가장 높은 점수를 받았는지 정리한 표입니다.
- `model_strength_summary.csv`: 공통 XAI 방법만 사용해 모델별 평균 지표를 계산한 표입니다.
- `method_strength_summary.csv`: 모델-방법 조합별 지표입니다. 공통 방법이 아닌 LIME도 참고용으로 포함됩니다.
- `method_agreement_summary.csv`: 같은 모델 안에서 여러 XAI 방법이 얼마나 일관된 설명을 내는지 정리한 표입니다.
- `model_strength_report.md`: 사람이 읽기 쉬운 최종 요약 보고서입니다.
- `model_strength_report.json`: 위 내용을 프로그램에서 다시 활용하기 위한 구조화된 결과입니다.
